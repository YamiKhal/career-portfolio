A platform that allows users to **log their gaming** journey, generate **fun numbers** in a **gamified, fun way**.

Why was it created? Why is it designed the way it is? This is what I will be answering.

## The problem

[Backlogged](https://backloggd.com/) is a **minimal** game tracking platform with easy to use features. 
[Infinite Backlog](https://infinitebacklog.net/) provides more **complicated** features with more complex UI. 

I wanted to create an application that **merges both platforms**.

With the minimal Backlogged UI kit and the extensive gamified features Infinite Backlog provides, TrackGames is meant to be the middle point between these two platforms.

## Design goals

TrackGames is built around a few guiding principles that I use while building new features.

- **Accessible UI**:
    Features implemented should stay easy to discover and intuitive.
- **Rewarding experience**:
    The more users interact with features, make numbers go up, showcase milestones, etc.
- **Related info**:
    Information should appear where users expect it, avoiding extra pages.
- **Consistent systems**:
    Similar interactions should function and display the same, ensuring the experience remains predictable.

## Data flow

Instead of requesting data from external services every time a page loads, TrackGames stores its own version of the data locally and updates it when needed.

> TrackGames uses two primary data sources: [IGDB](https://www.igdb.com/) for game metadata and **Google Analytics** for platform usage statistics. TrackGames controls how the data is stored and presented throughout the application.

This approach improves response times, **avoids external API limitations** and allows additional information to be attached to enrich the platform's features.

## Organizing information

With reliable game and user data available, TrackGames combines information from multiple sources into a single page. 

Rather than spreading related information across several views, the goal is to keep everything users need in one place.

## Customization

Customization is a key element I wanted to provide the users of the platform.

- Background images
- User titles
- Custom game lists
- Markdown widgets
- Showcasing profile stats

Markdown is the most flexible customization feature. Users can create layouts, embed images and videos, link to other pages, and personalize their profiles **without writing HTML**.

> with a URL scanning layer to ensure nothing unsafe is posted on users' profiles. If anything passes that, there is also a **report button**.

With this, users have the tools to showcase their stats, their history and what they are about as gamers, which is the ultimate goal of the platform.

## Technology

TrackGames is built with **Next.js**, **TypeScript**, **PostgreSQL**, **Prisma**, **Redis**, and **Docker**. Each technology was chosen to solve a specific problem that I faced during development.

**Next.js** provides server-side rendering and API routes. **TypeScript** improves maintainability by catching mistakes early and making larger systems easier to extend.

For persistent storage, **PostgreSQL** manages the application's relational data, while **Prisma** provides a type-safe interface for querying and updating the database.

**Redis** is used as a caching layer for frequently requested data, reducing unnecessary database queries and improving response times. **Docker** keeps development and deployment environments consistent for the sake of less development headaches.

**Coolify** is used to host the website on a VPS obtained from Hetzner, publicly available under the [trackgames.app](https://trackgames.app) domain.

## Future work

By no means is TrackGames a finished product. It's still far behind what I envision the platform to be.

I want to add more gamified features inspired by Infinite Backlog, add more tools that would allow me to track how well each feature is doing since the platform is shipped, and continue to monitor ease of use since that's a main goal.

- **Achievement progression**: The ability to sync and track achievements of different gaming platforms like steam or PSN.
- **Improved social interactions**: Nominate games for platform awards, form groups, game together with shared and dedicated group gaming systems.
- **Recommendation engine**: Personalize the main page a bit for users depending on what they have in their library.

These are only a few examples, more are still getting refined in my head.

I want TrackGames to garner a community, allow the community to interact from within as much as possible and also pay attention to what they want, to make the perfect tracker community platform.

## Conclusion

**TrackGames was not built to compete** by adding every single feature out there. Its goal is to become the game tracker I personally want to use.

The goal has always been to find the middle ground between powerful features and a clean interface. If other players enjoy using it as much as I enjoy building it, then TrackGames has achieved exactly what it set out to do.
